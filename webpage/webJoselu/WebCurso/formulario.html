Formulario




Mujer Hombre

Los formularios nos permiten enviar datos al servidor (BBDD y otras fuentes de almacenamiento).


El elemento "input" tiene un montón de atributos:


  • text

  • email

  • search

  • url

  • tel

  • number (junto con min, max y step)

  • range (junto con min, max y step)

  • date

  • week

  • month

  • time

  • datetime

Con estos atributos las casillas ya llevan incluídos los elementos de validación pertinentes a cada uso.


Esto facilita enormemente el diseño del formulario ya que no tenemos que incluir, como antigüamente, JavaScript de validación.

Tips



El type "tel" no hace ninguna validación. Informa al navegador de que "lo que hay ahí" es un teléfono y lo adapte dependiendodel dispositivo. No es lo mismo verlo en un navegador de tablet, de móvil etc.


"placeholder" nos permite añadir un comentario en la casilla a rellenar indicando qué debe introducirse en ella.


"required" indica al navegador que este campo de de relleno obligatorio.


"autofocus": la función de este atributo es poner el cursor de manera activa en un input del formulario sin necesidad de hacer click en él.


"form": con este atributo (no confundir con la etiqueta) no es obligatorio que los input de un formulario estén dentro del mismo.


"pattern": mediante este atributo podremos delimitar mediante expresiones regulares el contenido insertado por el usuario en los inputs de los formularios.


"multiple":con este atributo se pueden seleccionar varios ficheros o emails en el envío de formularios, es compatible con los input de tipo file o email. Los datos los separaremos con comas.


La verdadera potencia de "datalist" radica en la posibilidad de llenar las listas desde BBDD etc.


Con las especificaciones HTML5 hemos visto una serie de atributos que son muy útiles para hacer validaciones automáticas. Sin embargo para hacer validaciones más complejas estas utilidades se nos quedan cortas.


La API forms de HTML5 nos provee de varios elementos que nos permitirán hacer estas validaciones:


  • setCustomValidity (método)

  • invalid (evento)

  • validityState (objeto)
