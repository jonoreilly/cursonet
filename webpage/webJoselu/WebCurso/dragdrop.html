Drag & Drop

Eventos de la API Drag&Drop para el objeto a arrastrar:


  • dragstart: desencadena la acción al inicio del arrastre.

  • drag: desencadena la acción durante el proceso de arrastre.

  • dragend: desencadena la acción al terminar de arrastrar.



Eventos de la API Drag&Drop para el objeto de destino:


  • dragenter: desencadena la acción cuando el puntero del ratón entra en el área de destino.

  • dragover : desencadena la acción cuando el puntero del ratón se encuentra sobre el área de destino.

  • drop: desencadena la acción al soltar el objeto arrastrado sobre el área de destino.

  • dragleave: desencadena la acción cuando el objeto arrastrado abandona el área de destino.



Los métodos del objeto dataTransfer son dos:


  • setData: en donde declaramos qué datos son transferidos.

  • getData : en donde declaramos qué datos serán capturados.


Tips



La API Drag & Drop forma parte del standard HTML5.


Consiste en "coger" con el ratón un elemento de una localización en nuestra web y sin soltarlo arrastrarlo hasta la posición deseada.


Todos los elementos son "arrastrables".


En la acción participarán dos objetos: el objeto que se arrastra y el objeto de destino.


Son posibles varios objetos de destino.


El método preventDefault() resetea el comportamiento del navegador con respecto a arrastrar objetos (cada navegador hace una cosa distinta...). Esto nos da via libre para decidir nosotros, con nuestro código, cómo se ha de comportar cuando arrastremos un objeto.



Para que al arrastrar una imagen se vea la "ghost image" el escrtorio debe estar con el tema "aero". Con el tema "clásico" es posible que no se vea. Esto también depende de navegadores etc.
