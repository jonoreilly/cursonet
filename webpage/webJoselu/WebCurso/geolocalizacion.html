Geolocation - Métodos

Éstos son los métodos mas importantes de la API Geolocation:


  • getCurrentPosition(ubicación, errores, configuración)

  • watchPosition(ubicación, errores, configuración)

  • clearWatch(ubicación, errores, configuración)


Tanto "errores" como "configuración" son opcionales.


getCurrentPosition nos da la posición solo en el momento en que la pedimos.


watchPosition funciona igual que getCurrentPosition pero ésta tiene la posibilidad de refrescar los datos cada cierto tiempo. Si el usuario se mueve, por ejemplo con un dispositivo movil, nos refrescará la posición.


clearWatch "resetea" el watchPosition indicando que deje de dar la ubicación del usuario.


Tips



Para que esta API funcione en nuestros navegadores es necesario activarla, ya que por defecto viene desactivada. En Google Chrome: configuración --> configuración avanzada --> Privacidad y seguridad --> Configuración de contenido --> Ubicación --> Permitir


Geolocation es un objeto que pertenece a su vez al objeto Navigator.

API Geolocation - Objetos retornados

Tanto getCurrentPosition como watchPosition retornan un objeto "Position" siempre que la localización tenga éxito.


Este objeto "position" lo recibe la "CallBack" ("ubicación" en nuestro ejemplo anterior) como argumento.


Propiedades del objeto "Position":


  • Coords: contiene el conjunto de valores que determinan la posición.

    Esta propiedad tiene una serie de valores:

    • Latitude

    • Longitude

    • Altitude

    • Accuracy

    • altitudeAccuracy

    • Heading

    • Speed


  • Timestamp: propiedad que indica el momento exacto en el que se determinó la posición.

Tanto getCurrentPosition como watchPosition retornan un objeto "Error" siempre que la localización haya fallado.


Este objeto "error" lo recibe la "CallBack" ("errores" en nuestro ejemplo anterior) como argumento.


Propiedades del objeto "Error":


  • Code: representado por 3 constantes:

    • 1: Permiso denegado.

    • 2: Ubicación no disponible.

    • 3: Tiempo para detectar ubicación excedido.


  • Message: Dependiendo del error generado.

El tercer parámetro: configuración

Este parámetro está especialmente diseñado para dispositivos móviles.


Este parámetro, al contrario de los dos primeros que son CallBacks, puede contener un objeto literal.


Este objeto literal puede configurar tres propiedades:


  • enableHighAccuracy: booleano. Si es true se utilizarán sistemas GPS para la localización. Por defecto es false.

  • Timeout: tiempo en milisegundos para llevar a cabo la localización. Si no se obtiene, devuelve "timeout".

  • maximumAge: determina si se hace uso de la caché para buscar la última localización.

Tips



El método "watchPosition" consume muchísimos recursos en nuestro dispositivo móvil.


Un objeto literal podría ser (por ejemplo):


var configuracion ={enableHighAccuracy:true, Timeout:2000, maximumAge:6000 }
